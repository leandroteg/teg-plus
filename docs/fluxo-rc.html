Etapa de decisão Aprovado Esclarecimento / devolução (volta atrás) Rejeitado (fim) Fase operacional Elo CD → Compras (implementado)
  • Etapa CD (triagem): a Solicitação de Material entra como aberta; o triador do CD (lotação com faz_triagem) aprova, separa e atende pelo estoque (baixa de saldo) — total (atendida) ou parcial. Só o que falta em estoque deveria seguir para Compras.
  • NOVO Elo CD → Compras (implementado): botão "Encaminhar p/ Compras" na triagem chama o RPC est_encaminhar_solicitacao_compras, que gera a RC (em_aprovacao, com aprovação pendente) dos itens sem saldo e marca a solicitação como encaminhada_compras (vínculo cmp_requisicao_id).
  • Gap conhecido: "Atender pelo estoque" exige base_destino_id na solicitação, mas o formulário "Nova Solicitação" ainda não captura a base de destino → atender falha até isso ser corrigido.
  • NOVO Esclarecimento técnico (em_esclarecimento) volta para a etapa Requisição; quem responde agora é o requisitante dono (antes só comprador), com push avisando.
  • NOVO Editar itens: o requisitante edita ao responder o esclarecimento; o aprovador edita direto em pendente/em_aprovacao ao pedir esclarecimento.
  • NOVO Antes/depois: qualquer edição de itens grava um registro de histórico, exibido na RC para todos (requisitante e comprador).
  • O esclarecimento da cotação (cotacao_em_esclarecimento) é respondido pelo comprador e volta para cotacao_enviada.
